# **🎮 바이브 코딩(Vibe Coding) 레트로 프리젠테이션 가이드**

이 프로젝트는 8비트 레트로 비디오 게임 콘셉트와 **Gemini 테마**를 결합하여 차세대 AI 프로그래밍 패러다임인 '바이브 코딩(Vibe Coding)'에 대해 소개하는 인터랙티브 웹 슬라이드 덱입니다. HTML 단일 파일로 구성되어 있으며, 웹 브라우저만 있으면 언제 어디서나 부드러운 애니메이션과 타이핑 효과가 적용된 발표를 진행할 수 있습니다.

## 

## **🕹️ 1\. 사용 방법 및 조작법 (Controls)**

웹 브라우저로 해당 HTML 파일을 연 후, 다음 키보드 단추나 마우스 클릭을 이용해 프레젠테이션을 제어할 수 있습니다.

### **⌨️ 키보드 단축키**

* **다음 슬라이드 이동**: 오른쪽 화살표(▶) 또는 스페이스바(Space)  
* **이전 슬라이드 이동**: 왼쪽 화살표(◀)  
* **스테이지 셀렉트 매트릭스 토글**: H 또는 h (원하는 슬라이드로 즉시 워프 가능한 대시보드 창이 열립니다.)

### **🖱️ 마우스 조작**

* **화면 클릭**: 다음 슬라이드로 이동합니다. (텍스트나 버튼 링크 제외)  
* **대시보드 카드 클릭**: H 키를 눌러 활성화된 매트릭스 창에서 원하는 스테이지 카드를 클릭하면 해당 슬라이드로 바로 이동합니다.

## 

## 

## **📊 2\. 인터페이스 및 주요 기능 (Features)**

* **성장형 HUD 보드**: 상단 인터페이스에 게임 스코어(SCORE)와 레벨업 게이지(LEVEL UP), 현재 스테이지(STAGE) 정보가 표시됩니다. 슬라이드를 넘길 때마다 스코어가 750점씩 누적되며 경험치 바가 차오릅니다.  
* **터미널 타이핑 이펙트**: 각 슬라이드가 활성화될 때마다 실제 레트로 PC 터미널에서 글자가 입력되듯 커서가 깜빡이며 텍스트가 신속하게 출력됩니다.  
* **순수 CSS 픽셀 아트**: AI 에이전트를 상징하는 Gemini 아이콘과 노트북 아이콘이 순수 CSS 복셀(픽셀 그레이디언트) 애니메이션으로 구현되어 화면 양옆에서 둥실둥실 떠다닙니다.

## **📝 3\. 슬라이드 상세 구성 (Slide Index)**

프레젠테이션은 총 15개의 스테이지(인트로, 엔딩 포함)로 구성되어 있으며, 바이브 코딩의 태동부터 실태, 명과 암, 그리고 인프라적 대안까지 깊이 있는 인사이트를 다룹니다.

| 스테이지 | 타이틀 (주제) | 핵심 내용 |
| :---- | :---- | :---- |
| **INTRO** | **VIBE CODING LEVEL 01** | 타이틀 표지 및 Gemini 마스코트 연출 |
| **STAGE 01** | **바이브 코딩의 정의와 개요** | LLM 프롬프트를 통한 프로그래밍과 설계 역량의 중요성 |
| **STAGE 02** | **바이브 코딩의 탄생과 유래** | 안드레이 카파시의 용어 정립 및 메리엄-웹스터 사전 등재 |
| **STAGE 03** | **안드레이 카파시의 비전** | "가장 인기 있는 프로그래밍 언어는 영어" 개념 설명 |
| **STAGE 04** | **타이핑 보조 도구 vs 바이브 코딩** | 코드 구조의 완전한 이해 유무에 따른 개념 차이 정의 |
| **STAGE 05** | **비개발자의 실험: '한 사람을 위한 소프트웨어'** | NYT 케빈 루스 기자의 소규모 맞춤형 앱 빌딩 실험 |
| **STAGE 06** | **실제 구현 사례: '런치박스 버디'** | 자투리 식재료 카메라 분석 맞춤형 식단 제안 앱 사례 |
| **STAGE 07** | **비개발자 코딩의 한계와 비판적 시각** | 개리 마커스의 지적 (오픈소스 조합 모방, 보안 리스크) |
| **STAGE 08** | **실리콘밸리 스타트업의 이용 실태** | Y 콤비네이터 분석 지표 (스타트업 25%가 코드 95% 이상을 AI로 빌드) |
| **STAGE 09** | **전문적 환경에서의 위험성: 유지 보수의 부재** | 레거시 시스템 이식 시 발생하는 보안 결함 누적 문제 |
| **STAGE 10** | **바이브 코딩의 아이러니: AI의 생성 거부** | 아스 테크니카 보고 (AI 어시스턴트가 유저에게 코딩 공부를 권하는 현상) |
| **STAGE 11** | **인프라적 장벽: 웹 배포의 한계** | 로컬 작동 코드를 웹 서버 및 도메인에 안착시키는 고전적 지식의 필요성 |
| **STAGE 12** | **배포 패러다임 혁신: 대화형 PaaS의 진화** | Vercel, Netlify 및 모델 컨텍스트 프로토콜(MCP) 에이전트 통합 플랫폼의 성장 |
| **STAGE 13** | **결론 및 시사점: 소프트웨어 개발의 미래** | 소프트웨어 생산의 민주화 속 인간 리더의 '도메인 전문성' 가치 상승 |
| **CLEAR** | **STAGE CLEAR\!\!** | 프리젠테이션 종료 및 마침 인사 |
